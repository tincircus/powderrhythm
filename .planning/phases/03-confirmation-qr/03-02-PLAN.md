    Route implementation:
    - Extract uuid from req.params
    - Query DB: db('tickets').select('uuid').where({ uuid, status: 'confirmed' }).first()
      NOTE: select only 'uuid' column — no PII (buyer_name, buyer_email) needed here
    - If no ticket: return res.status(404).end()
    - Build qrUrl: const APP_URL = process.env.APP_URL || 'http://localhost:3000'; then `${APP_URL}/ticket/${ticket.uuid}`

    - Stream PNG: QRCode.toFileStream(res, qrUrl, { width: 300, margin: 2, errorCorrectionLevel: 'M' })
    - Attach stream error handler on the return value of toFileStream:
      .on('error', (err) => { console.error('QR stream error:', err); if (!res.headersSent) res.status(500).end(); })
      NOTE: Express 5 global error handler does NOT catch stream errors — this .on('error') is required
      to prevent server crashes when the buyer navigates away mid-stream

    Express 5: no try/catch needed for the DB call — async errors auto-forwarded.
    Keep all existing route handlers (pending, api/ticket-status, /ticket/:uuid) intact.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → GET /ticket/:uuid/qr.png | UUID from URL params — untrusted string passed to Knex WHERE |
| res stream | PNG bytes piped directly to response — stream error must not crash server |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Information Disclosure | GET /ticket/:uuid/qr.png | accept | UUID v4 entropy (122-bit) makes enumeration infeasible at 50-seat scale. Rate limiting deferred to Phase 6 (SEC-04). |
| T-03-06 | Denial of Service | toFileStream stream error | mitigate | Attach .on('error') handler to toFileStream return value; guard with !res.headersSent before sending 500. Prevents server crash when buyer navigates away mid-stream. |
| T-03-07 | Spoofing | QR URL encoding APP_URL | accept | APP_URL is operator-controlled env var. If unset, defaults to http://localhost:3000 (safe dev fallback). Production deployment must set APP_URL to https://tickets.powderrhythm.com. |
| T-03-08 | Information Disclosure | Content-Disposition filename | accept | Filename is "ticket-<uuid>.png" — UUID already in URL, no new information leaked. |
</threat_model>
